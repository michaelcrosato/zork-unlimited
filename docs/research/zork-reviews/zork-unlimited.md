## Verdict

This experiment is the family's **methodology flagship**: a hand-authored 59KB build spec (`ADVENTUREFORGE_BUILD_SPEC.md`) executed by Claude Code/Opus 4.8 ‚Äî all five engine stages (deterministic core ‚Üí CYOA ‚Üí parser ‚Üí Sierra-scoring ‚Üí RPG ‚Üí React UI) built green in **under 4 hours on day 1** (13:58‚Äì17:42 May 31), after which the remaining 9.5 days were an autonomous AFK improvement loop that authored 40+ packs and fixed 330+ logged bugs. Its three durable contributions are (1) the **two-mode testing model** ‚Äî deterministic validators + exhaustive BFS solver prove structure, a mandatory per-cycle **blind LLM playtest over MCP** judges experience; (2) the **anti-reward-hacking verifier-integrity gate** (`scripts/verify-integrity.ts`) that hard-blocks test weakening and hash laundering; (3) the most honest **harness-pathology self-diagnosis** in the family ‚Äî five ultraplan docs that name, with measured evidence, the polish treadmill, redundant multi-agent fan-out, the self-raising-target trap, orphaned workers, and 256M-token/12h cost mechanics. The engine itself is small (800-line core), genuinely deterministic, and reusable; the weakest claim is that the spec's writer‚Üíadapter authoring pipeline was only ever exercised by a mock ‚Äî real content came from the loop agent writing YAML directly.

## 1. Origin & mission

- **The original prompt is preserved verbatim**: `ADVENTUREFORGE_BUILD_SPEC.md` (compiled 2026-05-31, 20 sections, 1,064 lines). Goal sentence: *"Prove, end to end, that an AI can author a text adventure, compile it into a schema-valid game, run it on a deterministic headless engine, play it through a structured action API, test it, record its experience, find design/logic flaws, fix them, and lock the fix with a regression test."* Intended reader: *"a frontier coding agent (e.g. Claude Code with Opus 4.8, Codex with GPT-5.5, or Gemini 3.5 Flash via Antigravity) plus its human supervisor."* This is the most complete spec in the four-repo family ‚Äî it cites RPGBench, TALES, Jericho, TextWorld, TextQuests as the evidence base for "LLMs are excellent content generators and poor rule engines."
- **Governance pivoted mid-run.** The spec originally required a human-approval gate (¬ß0) and a six-item engine-extension gate (¬ß14). An "AUTHORITATIVE OVERRIDE ‚Äî TRUST, BUT VERIFY" header was prepended and `AGENTS.md` became the charter: agent gets *free rein over everything*; the automated verification bar (`npm run health` + `verify:integrity`) is the only gate. Core principle, stated and enforced: **"don't route around the verifier."**
- **Timeline**: May 31 ‚Äî Stages 0‚Äì5 + PRs #1‚Äì5 + first AFK cycle commit by 22:50. Jun 1 ‚Äî peak 105 commits. Jun 5 ‚Äî token tuning (loop default ‚Üí Sonnet, state-log rotation, dot reporter). Jun 6 ‚Äî zero commits (loop down). Jun 7 17:58 ‚Äî owner injected a **"non-technical VP pop quiz"** framing into `AI_AGENT_PROMPT.md` (commit messages must be plain-English, outcome-led ‚Äî a deliberate prompt-layer experiment). Jun 7‚Äì8 ‚Äî stale-description fix treadmill, then 16-pack authoring burst. Jun 8‚Äì9 ‚Äî three ultraplans, stand-down over `agy`-vs-claude `main` ownership, final commit = "orphaned-cycle rescue" of the 11th RPG pack.

## 2. Engine design

- **Three-layer architecture** (¬ß3): Story (prose) ‚Üí Game-design content pack (schema-valid YAML) ‚Üí Engine (pure code). The Layer-2/3 boundary ‚Äî *content is validated data, never code* ‚Äî is the system's central invariant and held for all 10 days.
- **Deterministic core** (`src/core/`, ~800 lines): one public pure reducer `step(state, action) ‚Üí {state, events, ok, rejectionReason}`; seeded PRNG `rngForStep(seed, step)` (no `Math.random`/clock anywhere); canonical-JSON SHA-256 state hash; saves bound to content hash (mismatch = hard error); traces replay with per-step hashes and `divergedAtStep`. Determinism is a fast-check **property test** (random action sequences run twice ‚áí byte-identical traces), plus purity, save/load round-trip, and legality-agreement properties.
- **One state model, four modes**: unified `GameState` (`flags`, `vars`, `inventory`, `objectState`, `journal`, `questStage`‚Ä¶); modes add fields, never replace. RPG schema is literally `ParserPackSchema.extend({ enemies })`; the RPG runner = parser runner + `ATTACK` + `skill_check` (d20 + var vs DC, every roll through the PRNG so fights replay exactly). Closed condition/effect DSLs ‚Äî content cannot invent verbs. `docs/stage4_rpg_gate.md` records the exemplary six-item extension bundle (spec, schema, unit tests, scenario test, backward-compat hash assertion, fresh trace).
- **Jericho-style legal-action API**: the model never sees a raw parser; every observation enumerates `available_actions` (id + command + structured action). Exposed via an **MCP server** (22 tools, `src/mcp/tools.ts`, 924 lines; `.mcp.json` checked in) so any harness plays via native tool calls; mode auto-detected from pack structure.
- **Validation**: per-mode validators with documented conservative approximations ‚Äî CYOA full-graph proofs (reachability, ending reachability, soft-locks, flag/item feasibility, contradictions); parser adds key-satisfiability, item-obtainability fixpoint, `quest_critical` permanent-loss guard, dialogue termination; RPG adds combat-winnable/skill-passable. `content/broken-fixtures/` are negative packs that MUST fail with specific codes. An **exhaustive BFS solver** proves every ending reachable, no reachable soft-lock, and score economy soundness (reachable max == declared `max_score`).
- **Procedural eval distribution**: `src/gen/{cyoa,rpg}_generator.ts` mint byte-identical packs from seeds; `corpus/` holds 12 sealed generated packs with pinned hashes (`corpus/manifest.json`). Rationale (README): *"A frozen eval set‚Ä¶ stops being a moving target and becomes a memorisable one."* This is a genuinely novel idea for self-improving loops.
- **React UI** (`ui/`) is a pure view: compiles packs in-browser, drives the identical `step` reducer (pure-JS SHA-256 for byte-identical browser hashes); never decides legality.
- **Well-designed**: the determinism spine, mode-as-thin-skin reuse, conservative-approximation honesty in validators. **Accidental/weak**: the `health` npm script is a 43-clause `&&` chain of per-pack validates; the 40KB 22-tool MCP surface grew unbounded; README pack table froze at 17 while reality reached 43.

## 3. Content & generation pipeline

- **43 packs** at snapshot: 18 CYOA / 14 parser / **11 RPG** (`printers_night` rescued in the final commit). All validate 0 errors/0 warnings; each pack auto-discovers ~10 tests via `tests/regression/all_packs_validated_by_bar.test.ts`.
- **A distinctive house style emerged unprompted**: 1690s English historical-procedural investigation fiction ‚Äî alnager (cloth inspection), tidewaiter, coroner's inquest, Goldsmiths' Hall assay (Great Recoinage 1695), press-gang impressment, will forgery with paleographic evidence. Each authoring cycle logged claimed novelty: new stats (`authority`, `cunning`, `physick`, `rhetoric`, `stealth`), new verbs (`TOUCH`, `COMPARE`), new topologies ("two-step physical‚Üíchemical test", "evidence-branching convergent wins"), standardized 50-point score economies with combat-fallback paths capped at 40/50.
- **Critical caveat (from `docs/ULTRAPLAN-2026-06-02.md`)**: *"The ¬ß1 machine itself has only ever been exercised by a mock."* The spec's writer‚Üíadapter‚Üívalidator pipeline (`agents/authoring/`, `content/engine_contract.yaml`, beat classification `fully_supported ‚Ä¶ requires_engine_extension`) exists and is tested, but `MockAuthorProvider` was hardcoded in the MCP path; the loop agent wrote pack YAML directly instead. Real-provider adapters exist behind env vars but a keyed end-to-end authoring run never happened.
- **`docs/RPG-STANDARDIZATION-PLAN.md`** is the best content-architecture doc: a mechanic√ómode first-class matrix (vars/variants/death/quest_stage/score/skill_check/deadline/combat), a per-story appropriateness rubric (moral-fork CYOA gets no scoring ‚Äî "you don't score morality"), and the keystone reframe **"RPG ‚âÝ combat"** (`breaking_weir` is a combat-less skill-chain RPG). Recommended "uniform palette, enabled per fiction" over mass mode conversion.
- **`docs/stage4_rpg_gate.md`**: hidden-var conventions (`__enemy_hp_<id>`), no-counterattack-on-killing-blow edge cases, backward-compat content-hash pinning ‚Äî a model record for adding mechanics to a deterministic engine.

## 4. AI blind-playtesting methodology (the family's unique research)

- **Protocol** (`docs/blind_playtest_protocol.md`): spawn a **fresh subagent with NO repo access** that plays only through `mcp__adventureforge__*` tools; verbatim locked prompt (objective / 6-section report format / tool guidance / boundary "DO NOT read, open, grep, or cat ANY repo files"); 2‚Äì3 playthroughs; report = route log, mechanical soundness, **clarity 1‚Äì5 + enjoyment 1‚Äì5**, confusion points, bugs-with-scene, verdict. Findings triaged to one fix layer (`content | hint_text | quest_structure | engine_rule | validator | test`), then locked with a `traces/bugs/` artifact + regression test.
- **Standalone harness** (`blind-tester/run.sh`): runs `claude -p` from an **isolated temp cwd** with `--disallowedTools Read Edit Write Bash Glob Grep WebFetch WebSearch Task`, `--strict-mcp-config` (only the game server reachable), `--permission-mode bypassPermissions`, on **subscription (no API key)**. `smoke.mjs` proves the MCP plumbing token-free. `BLIND_AGENT_CMD` seam makes it provider-agnostic (`gemini -p` example documented) ‚Äî i.e., it is already a **cross-model playtest instrument**. Two-level blindness doctrine: interface-level (done) + data-level observation masking (deferred; credited to sibling repo 3).
- **Enforcement**: `require_playtest_record` in `loop.sh` ‚Äî no blind-playtest report on disk ‚áí no commit, every cycle. Honest limit acknowledged: it proves a report exists, not that a model truly played.
- **Demonstrated value**: first run on *Watchtower Road* surfaced 4 polish findings ‚Üí `bug_0002` + regression; the loop then found ~one real bug per cycle for 30+ cycles (the bug_0282‚Äì0331 stale-reactive-description family); late-run reports consistently scored packs 4‚Äì5/5 and still produced actionable findings (e.g., "hold_off is an infinite loop with no state change"). Known flaw, fixed only on the last day: content_new cycles playtested the *baseline* pack, so ~16 new packs shipped structurally-proven but experience-unchecked ‚Äî the Jun 9 orchestration ultraplan redirected the playtest to the newly authored pack.

## 5. Loop/harness behavior & research

- **Pipeline** (`loop.sh`, 194 lines): `npm run ai:loop` (deterministic assessor emits `ai-runs/<id>/prompt.md`) ‚Üí `run_agent` (`claude -p --model sonnet --dangerously-skip-permissions`, prompt on stdin, `timeout` 2400s routine / 3600s ultraplan+content_new, SIGTERM‚ÜíSIGKILL) ‚Üí `npm run health` (blocking) ‚Üí `verify:integrity --against <pre-cycle-ref>` ‚Üí `require_playtest_record` ‚Üí commit/push. **Circuit breaker**: 5 consecutive no-progress cycles. `_revert_failed_cycle` resets failed scratch so one bad artifact can't wedge every later cycle. Project-scoped PID files because multiple identical-looking loops ran on the same machine (the four experiments!).
- **`scripts/verify-integrity.ts`** (701 lines) is the standout harness artifact: hard-blocks deleted/disabled tests (`.skip/.only/.todo`), test-count drops, protected-asset deletion, and hash re-pins with **no accompanying content change** ("the regenerate-to-make-red-go-green launder pattern"); allows-but-surfaces re-pins alongside real changes; includes a tautology-assertion detector (`expect(foo).toBe(foo)`). Cites EvilGenie (2511.21654), METR 2025-06, Anthropic 2025-11. Honest limit: catches mechanical, not semantic, weakening.
- **Assessor** (`src/afk/assessor.ts`, 821 lines): pure ranker over `content_fix / content_new / engine / repo` + 3 generator mint-and-check levers; LRU rotation of the blind-playtest target; `SATURATION_FLOOR = 0.5`; saturation triggers a bounded multi-agent **ultraplan** (8-cycle cooldown) that overwrites `docs/CURRENT_PLAN.md` and hands it to a fresh implementer subagent ("plan handed off as a document, not a context window").
- **An orchestrator layer existed above the loop**: an interactive Claude Code session that launched `loop.sh` in background, watched git/log/process state on wakeup ticks, and on anomaly paused‚Üídiagnosed‚Üífixed‚Üírelaunched (durable state in gitignored `ai-runs/orchestration-state.md`). Much of the late git history is orchestrator repair work.
- **Pathologies observed and diagnosed (all with fixes or rulings)**:
  1. **Polish treadmill** ‚Äî saturated assessor spent ~30 cycles fixing one *instance* per cycle of the same "stale reactive description" class (bugs 0282‚Äì0325). Fix: "catch the class" prompt nudge + Opus self-critique gate ("is this busywork?").
  2. **Redundant multi-agent fan-out** ‚Äî ultraplan re-aim #19 produced **six false alarms** (reviewers re-confirmed already-implemented features) because `CURRENT_PLAN.md` is overwritten. Fix: append-only **`docs/DECISION_LOG.md`** ledger; reviewers read it first and may not re-nominate "Confirmed CLOSED" items. Re-aims #20/#21 then logged "False alarms: None."
  3. **Self-raising-target trap** ‚Äî ultraplans raised `TARGET_PER_MODE` 3√ó (bug_0332/0335/0336), each raise consumed by an authoring burst that re-saturated. Ruled an explicit **üö´ ANTI-PATTERN** in DECISION_LOG: ceiling frozen at `{cyoa:20, parser:16, rpg:16}`; *"Pack COUNT is not the objective."*
  4. **Hung agent wedged the loop ~9h** ‚Äî circuit breaker only counted *completed* cycles. Fix: hard `timeout` on the agent turn.
  5. **Orphaned worker (critical, last day)** ‚Äî killing `loop.sh` + the `timeout bash -c` wrapper does NOT kill the `exec`'d `claude.exe` worker; it orphaned (parent=1), kept authoring, and would have committed past a pause. Fixed procedurally at the orchestrator; the loop.sh process-group fix was still backlog #1 at purge. The final commit is the rescue of that orphan's valid-but-uncommitted pack.
  6. **Token economics** (`TOKEN-REDUCTION-ULTRAPLAN` √ó2, measured): **256.6M tokens/12h** (~5.8M/cycle, 94% `cache_read`, ~71 round-trips/cycle √ó ~77‚Äì86k cached prefix). Governing equation: *cost = round-trips √ó prefix size*, not prompt size. Cost projections $337/day Sonnet, $562/day Opus; June 15 metered-billing change made it urgent. Ranked levers: PTC-script only the *fixed regression route* (scripting the exploratory route "would destroy the quality oracle"), measure-then-shrink the prefix (CLI already defers MCP schemas ‚Äî don't assume), Haiku for the mechanical blind playtest, terse-output defaults, 1-h cache TTL across 10s-apart cycles, and **in-repo token instrumentation first** ("the loop is cost-blind").
  7. **Two autonomous actors on one `main`** ‚Äî the late "agy" (Antigravity/Gemini) actor's contention crashed the claude loop twice (exit 255); `REVIEW_QUEUE.md` ends in a deliberate stand-down awaiting the owner's ownership call (options: agy keeps main / claude resumes / claude moves to a worktree branch).
- **Cadence**: 60/105/59/90/51/11/0/22/71/3 commits per day (May 31‚ÜíJun 9); 364 commits touched `AI_LOOP_STATE.md`; 75 bare "Autonomous AFK improvement cycle" commits plus descriptive per-change commits.

## 7. Pitfalls & anti-patterns

1. **Count-as-objective**: breadth targets (`TARGET_PER_MODE`) became a self-feeding fitness function the ultraplan kept raising. Quantity metrics in a self-improving loop get optimized; the loop itself had to legislate against it.
2. **Instance-fixing instead of class-fixing**: 30+ cycles on one bug family because the assessor had no above-floor signal for structural moves.
3. **Overwritten plans lose negative knowledge**: without an append-only ledger, multi-agent reviews re-derive settled questions at full fan-out cost (6 false alarms in one re-aim).
4. **The headline claim ran on a mock**: "AI authors a game" was demonstrated by the loop agent hand-writing YAML, not the instrumented writer‚Üíadapter pipeline with a real provider. Verify the load-bearing verb early.
5. **Process-tree hygiene on Windows/MSYS**: `timeout bash -c 'exec agent'` leaves an orphanable worker; a "stopped" loop kept committing. Need process-group kill + independent re-scan.
6. **Cost blindness**: no in-repo token telemetry; 94% of 21M tokens/hr was cache re-read driven by round-trip count ‚Äî invisible until manually aggregated from CLI transcripts.
7. **Quality oracle misaimed**: new content was playtested-by-proxy (baseline pack) for a full day of authoring; "valid" ‚âÝ "validated experience."
8. **Two autonomous writers, one branch**: crashes and a forced stand-down. One actor per branch; worktrees for coexistence.
9. **Self-graded novelty**: every authoring cycle ends "Not busywork" ‚Äî the self-critique gate is the right idea but is judged by the author-model; an independent judge would be stronger.
10. **Doc drift under autonomy** (README at 17 packs vs 43 real; stale verify-integrity docstrings flagged in DECISION_LOG but deferred repeatedly).

## 8. Carry-forward recommendations

**Seed the unified project with (engine):** the pure `step` reducer + seeded `rngForStep(seed, step)` + canonical-hash + content-hash-bound saves + trace replay with `divergedAtStep`; the closed condition/effect DSLs; the unified GameState with modes-as-extensions (`RpgPack = ParserPack + enemies` proves the shape); legal-action enumeration over MCP; validators with negative fixtures + the exhaustive BFS solver (endings reachable / no soft-locks / score economy); the seeded pack **generators + sealed corpus** as an anti-memorization eval distribution.

**Seed the methodology with:** the **two-mode testing model** (validators prove structure; blind LLM playtest is the only experience judge ‚Äî nothing in between); `blind-tester/` essentially as-is ‚Äî it is already provider-agnostic and is the natural **cross-model measurement instrument** for the controlled experiment (same packs, same seeds, swap `BLIND_AGENT_CMD`); `verify-integrity` as a standard harness gate (hard-block weakening, allow accompanied re-pins, tautology detection); the bug-artifact + regression-test discipline (¬ß15 format).

**Seed the harness rules with:** agent-turn `timeout` + circuit breaker + `_revert_failed_cycle` + PID-scoped stop + **process-group kill (fix the known hole first)**; mandatory quality-record gate before commit; append-only `DECISION_LOG.md` from day 1 (reviewer boundary); saturation‚Üíultraplan with cooldown rather than letting routine cycles grind; catch-the-class nudge + self-critique gate; per-cycle token instrumentation (`cost.json`) from cycle 1 ‚Äî this repo proves the loop is otherwise cost-blind; model-per-role (cheap model for mechanical playtests, frontier for the improvement agent); PTC/scripting only for non-reasoning fixed routes.

**For the controlled experiment design:** use `ADVENTUREFORGE_BUILD_SPEC.md` (with the trust-but-verify override decided *up front*, not pivoted mid-run) as the common brief ‚Äî it is self-contained, language-agnostic in ¬ß5‚Äì¬ß15, and names per-stage acceptance criteria that double as comparable checkpoints. Pin as controlled variables: model + effort, harness, loop params (`AI_LOOP_MODEL`, timeouts, `TARGET_PER_MODE` frozen, cooldowns), commit policy, and a **defined endpoint** (e.g., N packs per mode at fixed ceiling + blind-playtest score threshold) ‚Äî this run's open-endedness is what produced the treadmill. One autonomous actor per repo/branch, ever. Measure per-model: time-to-green-stage, packs/cycle, blind clarity/enjoyment trend, bug-class recurrence, tokens/cycle, and integrity-gate trips.

**Avoid:** pack-count fitness targets; playtesting proxies instead of the changed artifact; letting ultraplans edit their own incentive constants without an anti-pattern ledger; assuming harness internals (measure the prefix ‚Äî the 06-09 token doc caught the 06-08 doc's wrong assumption); and trusting "a strong model wrote it" anywhere the spec's ¬ß16 says not to.
